**② 반짝 전이 감지 코드 좌표 + "마운트 시 미발동" 근거**
- 상태/ref 선언: `src/components/GrapeBoard.tsx:402-403` — `prevRipeRef = useRef<boolean | null>(null)`(초기값 `null` = "미관측"), `justRipened` state.
- 감지 effect: `GrapeBoard.tsx:404-413`.
  ```
  const ripeNow = paceState ? paceState.ripe : null;
  if (prevRipeRef.current === false && ripeNow === true) {  // ← 전이 조건: 직전이 명시적 false였을 때만
    setJustRipened(true);
    setTimeout(() => setJustRipened(false), 700);            // 키프레임 0.7s와 동기화, 자동 원복
  }
  prevRipeRef.current = ripeNow;
  ```
- **마운트 시 미발동 근거**: 최초 렌더에서 `prevRipeRef.current`는 `null`이다. 마운트 시점에 이미 익어 있어 `ripeNow === true`여도 분기 조건은 `prevRipeRef.current === false`를 요구하므로(`null !== false`) 발동하지 않고, `prevRipeRef.current`만 `true`로 갱신되어 "관측됨" 상태로 넘어간다. 이후 실제로 `false`(익는 중)를 한 번 거친 뒤 `true`로 바뀌는 진짜 전이에서만 조건이 성립한다. FREE 보드 등 `paceState`가 없으면 `ripeNow=null`이라 조건이 성립하지 않고 ref도 `null`로 리셋돼 재관측 대기 상태를 유지한다.
- 렌더 배선: `showRipeSparkle={isNext && justRipened}`(`GrapeBoard.tsx:698`) → `GrapeCell`(`GrapeBoard.tsx:150`)이 `<span className="grape-ripe-sparkle" aria-hidden="true" />`를 GrapeSticker와 형제로 렌더(비소유 파일 `GrapeSticker.tsx`는 미수정 — wrapper div가 이미 버튼과 동일 기하라 별도 absolute 자식으로 충분).
