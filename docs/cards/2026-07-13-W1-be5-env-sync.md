## BE-5: .env.example ↔ 코드 ↔ PLAYBOOK env 대장 동기화 (B4)

- Severity: Low(온보딩) / 분류: 문서·설정 / 배정: haiku
- 필독: PLAYBOOK §5 환경변수 대장

### 소유 파일 (이 목록 밖 수정 시 반려)
- `.env.example`
- `docs/PLAYBOOK.md` (§5 표만)

### 문제/재현
코드가 참조하는 7개 변수가 `.env.example`에 누락 — 새 배포자가 크론·푸시를 조용히 비활성 상태로 띄우게 됨: `CRON_SECRET`, `VAPID_PRIVATE_KEY`, `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_SUBJECT`, `NEXT_PUBLIC_APP_URL`, `NEXT_PUBLIC_DEV_TOOLS`, `ENABLE_DEV_LOGIN`. PLAYBOOK §5 대장에는 `NEXT_PUBLIC_APP_URL`만 누락.

### 스펙 (시험 가능한 문장)
- `.env.example`에 7개 변수를 주석(한 줄 용도 + 미설정 시 결과)과 함께 추가. 값은 플레이스홀더(실제 시크릿 금지).
- `docs/PLAYBOOK.md` §5 표에 `NEXT_PUBLIC_APP_URL` 행 추가 (사용처: layout.tsx OG/shareCard 공유 URL, 미설정 시: 요청 origin 폴백).

### 제약
- 실제 시크릿 값 절대 금지 — 플레이스홀더만.
- PLAYBOOK은 §5 표 외 접근 금지.

### 검증법
```bash
# .env.example의 변수명이 코드 참조와 일치하는지:
grep -oE '^[A-Z_]+' .env.example | sort
```

### 보고 전 자가검증
각 주장을 이 세션의 도구 결과와 대조 — 증거를 가리킬 수 있는 작업만 보고. 검증 안 된 것은 "미검증" 명시.

### 산출
diff + 검증 로그. **커밋 금지** (git add도 금지 — 페이블이 수행).
