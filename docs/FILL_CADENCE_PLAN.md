## 1. 원칙 (전부 기존 정본에서 파생)

1. **처벌 금지** — 늦음·몰아침에 감점·경고·빨간색 없음(PRINCIPLES §1 따뜻함, ROADMAP Non-goals). 리뷰 NG-02(벌금 챌린지) 거절과 동일 축.
2. **진행 영원히 무료 불변** — 텀·숙성·보충은 전부 무료 기능. 수익화와 절대 결합 금지(스킨은 코스메틱만).
3. **아너 시스템** — 자기 기록 앱. 서버는 부정행위를 "막지" 않고 "기록"한다(소프트 모드 오버라이드 허용 + 플래그).
4. **기존 무파괴** — 기본값 FREE = 현행 동작 그대로. 기존 보드·기존 유저는 아무것도 달라지지 않는다.
5. **직렬 큐·낙관 채우기 불변** — `boardFillState.ts` 채움 파이프라인(낙관 반영→직렬 POST→화해)과 `isJustFilled` 600ms 연출 창은 건드리지 않는다. 텀 검사는 그 **앞단**(탭 허용 판정)과 **응답 메타**(paceState)로만 붙는다.

## 2. 텀 모델 4종

| 모드 | 의미 | 예시 습관 | 다음 알이 익는 시점 |
|------|------|----------|------------------|
| `FREE` (기본) | 제한 없음 — 현행 | 자유 기록 | 항상 익어 있음 |
| `DAILY_1` (추천 기본 제안) | 하루 1알 | 운동·기상·금연·필사 | 다음 "하루 경계"(§4) |
| `DAILY_N` | 하루 N알 (N=2~10) | 물 8잔·스트레칭 3회 | 당일 N개 미만이면 즉시, 채우면 다음 경계 |
| `WEEKLY_N` | 주 N알 (N=1~7) | 주 3회 헬스·주 1회 대청소 | 주간 N개 미만이면 즉시, 채우면 다음 주 시작 |

- 선택 위치: 보드 생성 4스텝 중 **크기 스텝에 "채우는 리듬" 서브 선택** 추가(스텝 수 불변 — 5스텝 확장 금지, 생성 마찰 최소화). 생성 후에도 보드 편집에서 변경 가능(단 하향 변경 시 이미 채운 알 소급 없음).
- 템플릿 연동: `templates.ts` 각 템플릿에 `recommendedCadence` **additive 필드**(제목·설명 등 기존 카피 불변). 템플릿 선택 시 텀이 자동 제안됨 — 운동=DAILY_1, 물마시기=DAILY_N(8), 주간 습관=WEEKLY_N.
- 릴레이 보드·선물 보드: v1에서는 **FREE 고정**(바톤·개봉 시맨틱과 간섭 차단). C4 이후 재검토.

## 3. 숙성 메커니즘 (간판 장치)

핵심: 잠금 화면이 아니라 **익어가는 화면**을 보여준다.

- **시각**: 다음 차례 알(현 `grape-next`)이 텀 대기 중이면 "익는 중" 상태 — 청포도 톤(leaf/lime 계열)에서 보라(grape 계열)로 채도가 시간 비례 전이. 기존 `grapeNextTension` 미세 펄스는 유지하되 색만 단계 전이. 완전히 익으면 현행 grape-next 모습 + 반짝(sunshine 토큰) 1회.
- **카피(신규 표면)**: 알 아래 보조 라벨 — "다음 알이 익고 있어요 · 내일 아침이면 제철" / 익으면 "제철이에요! 지금 따기 좋아요". (문구는 구현 카드에서 확정, 해요체)
- **소프트 가드(기본)**: 익기 전 알 탭 → 바텀시트 안내("아직 익는 중이에요") + **"그래도 채우기" 2탭 오버라이드**. 오버라이드 채움은 정상 채움과 동일하게 반영되고 `earlyFill` 플래그만 기록. 질책 문구 금지.
- **엄격 모드(옵트인)**: 보드 설정 `strictMode` — 오버라이드 버튼 제거, 익을 때까지 채움 불가. P06("어제 것 잠가달라")·자기구속 수요 대응. 기본 OFF.
- **보드 카드 상태**: 홈 보드 카드에 "오늘 몫 완료 🍇" 뉘앙스 표시(DAILY 계열에서 오늘치 채움 완료 시) — 오늘 할 일이 끝났다는 안도 신호. 리스트 과밀 방지 위해 배지 1개 이내.
- reduced-motion: 색 전이는 전역 `prefers-reduced-motion` 백스톱 대상(신규 키프레임 고유 이름 — CLAUDE.md 모션 규약).

## 4. 하루 경계 (GAP-03·GAP-13 공통 해소 기반)

| 항목 | 결정 |
|------|------|
| 기준 | **사용자 로컬 시간대** — `User.timezone`(IANA 문자열, additive, 기본 `Asia/Seoul`). 클라가 로그인/접속 시 갱신 |
| 새벽 리셋 | `User.dayResetHour`(0~6, 기본 0) — "하루의 시작 시각". 새벽 3시에 채워도 dayResetHour=4면 어제로 귀속. P14(새벽)·P22(3교대) 대응 |
| 적용 범위 | 텀 판정 + 스트릭 계산 + 시간캡슐 D-day 판정을 **같은 경계 함수로 통일**(현행 GAP-03 UTC 고정·GAP-13 자정 고정을 한 번에 치유) |
| 리스크 | 서버 판정 시각의 기준 단일화 필요 — 스티커는 UTC 저장 유지, 경계 계산만 timezone+resetHour로. 시간대 변경(여행) 시 스트릭 재계산은 하지 않음(관대한 쪽으로 고정) |

주의: 이 경계 함수는 통계(히트맵 날짜 귀속)와도 얽힌다 — C2에서 통계 반영 시 히트맵 버킷팅과 동일 함수 사용, 이중 구현 금지.

## 5. 놓친 날 장치 (관대함의 제도화)

- **보충 채우기(backfill)**: DAILY 계열에서 어제 몫을 못 채웠으면, 오늘 경계로부터 **48시간 이내 1알**에 한해 "어제 몫 채우기" 선택지 제공(익는 중 알 탭 시트에 노출). `Sticker.isBackfill=true`로 기록 — 통계에는 어제 날짜로 귀속.
- **스트릭 유예 연동**: 기존 `User.streakFreeze*` 필드(스키마 기존재, UI 미노출 — ROADMAP 백로그 "스트릭 유예 노출")와 합류. 보충으로도 못 살린 하루는 유예 1회 자동 제안. 두 백로그가 이 문서로 한 계보가 된다.
- 남용 방어: 보충은 연속 사용 불가(연속 2일 보충 시 3일째는 미제공) — 단 문구는 중립("이번엔 오늘 몫부터 채워볼까요").

## 6. 와이너리 연동 (숙성도 서사 — 신박 포인트)

완성 보드의 소믈리에 노트(cellarNote, 기존 기능)에 **빈티지 서사 1줄**을 자동 첨언:

| 채움 패턴(스티커 타임스탬프 분산) | 서사 예 |
|----------------------------------|--------|
| 텀 준수 높음·고른 분포 | "천천히, 고르게 익힌 빈티지" |
| 중간 공백 후 완주 | "쉬어간 만큼 깊어진 해" |
| 단기 집중 완주 | "한숨에 영근 여름 포도" |

- **비교·점수화 금지**: 등급·별점·"잘함/못함" 축 없음(Non-goal 랭킹 충돌 방지). 전 패턴이 긍정 서사 — 와인은 어느 해든 그 해의 맛.
- 표시 위치: 와이너리 병 상세·cellarNote 영역만. 홈·공유 카드에는 미노출(과시·비교 방지).

## 7. 알림 연동

- "포도알이 다 익었어요 🍇" opt-in 푸시 — **보드별 리마인더의 대체 옵션**으로 제공(리마인더 편집 화면에서 "시간 지정" vs "익으면 알림" 택1). 신규 알림 채널 증설 아님 → **알림 총량 불증**(ADHD 가드레일, P14·P21 근거).
- 기존 파이프라인 재사용: `NotificationSetting` 게이트 + DND + `src/lib/push.ts`. cron 판정에 §4 경계 함수 사용.

## 8. 데이터 레이어 (additive 전용 — 게이트 통과 설계)

```prisma
// 전부 additive·nullable/default — 기존 응답 계약 불변
Board.cadenceType   String  @default("FREE")   // FREE | DAILY_1 | DAILY_N | WEEKLY_N
Board.cadenceN      Int?                        // DAILY_N/WEEKLY_N의 N
Board.strictMode    Boolean @default(false)
Sticker.isBackfill  Boolean @default(false)
Sticker.earlyFill   Boolean @default(false)     // 소프트 오버라이드 기록
User.timezone       String  @default("Asia/Seoul")
User.dayResetHour   Int     @default(0)
// templates.ts(코드 상수): recommendedCadence 필드 추가 — DB 아님
```

- **API 계약**: `POST /boards/[id]/stickers` 응답에 `paceState`("ripe"|"early"|"backfill") **필드 추가만** — 기존 필드 제거·변경 없음. 소프트 모드 위반은 **200 + earlyFill 기록**(403/409 아님 — 낙관 채우기·직렬 큐·롤백 경로 불변). 엄격 모드 위반만 422 + 한국어 안내(클라가 선차단하므로 방어선).
- 마이그레이션: `npx prisma migrate dev` 단독 웨이브(PRINCIPLES §4 — 다른 작업과 병렬 금지).
- 검증 로직 위치: 서버 판정은 C2에서 도입. C1은 클라 판정만(서버는 기록 필드 수용) — 클라·서버 판정 불일치 창을 C1 릴리즈 노트에 명시.

## 11. KPI 연결 (WS1 이벤트 사전 등재분)

- `cadence_selected`(모드 분포) · `fill_early_override`(오버라이드율 — 텀 설계 적정성 신호) · `fill_backfill`(보충률) · **텀 준수율 = 리텐션 선행 지표 가설**(준수 유저의 D30 잔존 비교, WS1 가동 후 검증).
