포도알(newal) 수정 적용 미리보기

실제 앱 화면을 HTML로 충실히 재현해 현재(Before) 와 P0/P1 수정 적용(After) 을 나란히 놓았습니다. 색·카피·레이아웃은 실제 컴포넌트(page.tsx·ClayButton·BoardCard·GrapeBoard 등)에서 가져왔습니다. 실제 앱 코드는 한 줄도 수정하지 않았습니다 — 이건 목업입니다. 대비비는 추정치이며 적용 전 실제 렌더 색으로 재측정 권장.

Before — 현재 코드의 문제 After — 수정 적용 모습

1부 · 컴포넌트 Before / After (8종)

전 화면 공통 요소. 대부분 globals.css/tailwind.config.ts 한 곳 수정으로 앱 전체에 적용됩니다 (근본원인 R1·R2·R3·R8·R9).

1. 키보드 포커스 링R1

Tab으로 이동 시 어디에 포커스가 있는지 보이게. (8개 표면 공통 — .clay-button에 :focus-visible 추가 한 번)

Before :focus-visible 규칙 없음 → 키보드 포커스 표시 0. 직접 Tab 해보면 링이 안 생김.
After grape-glow 토큰 재사용한 :focus-visible 링. (이 버튼을 Tab/클릭 후 Tab 해보세요)
2. 입력 placeholder 대비R3

.clay-input::placeholder의 opacity:0.5 제거.

Before ≈ 2.1:1 ✗ (AA 4.5 미달) warm-sub #6E6680 × opacity .5. '8자리 이상' 규칙이 placeholder에만 있어 더 문제.
After ≈ 4.8:1 ✓ 불투명 전용 토큰. 규칙은 영구 helper로 승격 권장.
3. 캡션·FAB 대비R3

warm-light 본문 텍스트, 흰 '+' on grape-500.

Before 방금 · 나의 성장 기록 warm-light ≈ 2.6:1 ✗
+ FAB 흰'+' ≈ 2.1:1 ✗
After 방금 · 나의 성장 기록 warm-sub ≈ 4.9:1 ✓
+ grape-600 ≈ 3.1:1 ✓
4. 설정 토글 시맨틱R8

role=switch+aria-checked+라벨, on/off 비색 단서.

Before
효과음
div/button에 role·aria-label·aria-checked 없음 → SR은 '버튼'만, 상태는 색·위치로만.
After
효과음
role=switch + aria-checked + 라벨, 노브에 ON 텍스트(비색 단서).
5. 로딩 상태R7

보드 상세 로딩이 떠다니는 🍇 → 최종 형태 스켈레톤.

Before
🍇
포도판 불러오는 중...
맨 스피너급. 데이터 도착 시 레이아웃 점프.
After
보드 카드 형태에 맞춘 스켈레톤(.skeleton 이미 존재) → 점프 없음.
6. 파괴적 액션 확인R5

window.confirm() → in-app 확인 시트.

Before
이 페이지 내용:
정말 삭제할까요?
취소 확인
OS 블로킹 다이얼로그 — claymorphism·포커스 트랩·톤 전부 깨짐.
After
포도판을 삭제할까요?
삭제하면 되돌릴 수 없어요.
기존 모달 시스템(z-90) 재사용, 차분한 카피 + 파괴 액션 구분.
7. 포도알 그림자 틴트R11

시그니처 요소가 검정 그림자 → 보라-웜 틴트.

Before
.grape-empty·mini가 rgba(0,0,0,X) 검정 그림자 — 라벤더 면 위에서 회색빛.
After
rgba(94,63,128,X) 보라 틴트 — 면과 같은 색 계열로 통일.
8. 리워드 글로우 + reduced-motionR9·R2

box-shadow 무한 애니 → opacity 오버레이 + 모션 가드.

Before
🎁보상이 열렸어요
box-shadow를 무한 애니(repaint), reduced-motion에서도 계속 깜빡.
After
🎁보상이 열렸어요
블러 의사요소 opacity 애니(GPU) + prefers-reduced-motion 시 정지(이 페이지가 그 설정을 따름).

이건 목업입니다. 실제 React 앱을 렌더한 게 아니라, 실제 컴포넌트의 색·카피·레이아웃을 HTML로 재현한 Before/After 시각화입니다. 앱 코드는 수정하지 않았습니다. 대비비(예 2.1:1)는 추정치이며 적용 전 실제 렌더 색으로 contrast checker 재측정 권장. 키보드 포커스 링·reduced-motion·tabular-nums는 이 페이지에서 실제로 동작하므로 직접 Tab/시스템 모션설정으로 확인 가능. 전체 근본원인·90개 항목은 같은 폴더 REPORT.md·preview.html 참고.
